Система контроля версий Git

Git-scm.com

Начало работы

Инициализация репозитория

mkdir test-repo
cd test-repo
git init

Настройка автора изменений

git config --global user.name "Ivan Belyavtsev"
git config --global user.email djbelyak@gmail.com

Работа над текущими изменениями (шаг 1)

Сделать изменение:

$ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

        modified:   Readme.md

no changes added to commit (use "git add" and/or "git commit -a")

Работа над текущими изменениями (шаг 2)

Добавление файлов в индекс репозитория

$ git add Readme.md
$ git status
On branch master
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

        modified:   Readme.md

Работа над текущими изменениями (шаг 3)

Фиксация изменения

$ git commit -m "Added important information"
[master 31f9a86] Added important information
 Committer: Ivan Belyavtsev <djbelyak@gmail.com>


 1 file changed, 3 insertions(+), 1 deletion(-)

Из чего состоит коммит

$ git show HEAD
commit 31f9a868ae30b289034d1ddff7379775ddd7b810 (HEAD -> master)
Author: Ivan Belyavtsev <djbelyak@gmail.com>
Date:   Mon Oct 22 20:15:07 2018 +0300

    Add important information

diff --git a/Readme.md b/Readme.md
index e007a9a..55627b8 100644
--- a/Readme.md
+++ b/Readme.md
@@ -1 +1,3 @@
-# Прочти меня
\ No newline at end of file
+# Прочти меня
+
+Тут важная информация
\ No newline at end of file

Как посмотреть историю коммитов

$ git log
commit 31f9a868ae30b289034d1ddff7379775ddd7b810 (HEAD -> master)
Author: Ivan Belyavtsev <djbelyak@gmail.com>
Date:   Mon Oct 22 20:15:07 2018 +0300

    Add important information

commit 953d1e02239a8012607c2c89d0c61932424e6a95
Author: Ivan Belyavtsev <djbelyak@gmail.com>
Date:   Mon Oct 22 19:35:45 2018 +0300

    Init commit

Вернуть рабочую копию к определенному состоянию

$ git checkout HEAD~1
Note: checking out 'HEAD~1'.

You are in 'detached HEAD' state. You can look around, make experimental
changes and commit them, and you can discard any commits you make in this
state without impacting any branches by performing another checkout.

If you want to create a new branch to retain commits you create, you may
do so (now or later) by using -b with the checkout command again. Example:

  git checkout -b <new-branch-name>

HEAD is now at 953d1e0 Init commit

Вернуться к крайнему состоянию

$ git checkout master
Previous HEAD position was 953d1e0 Init commit
Switched to branch 'master'

Что будет, если сделать новый коммит в detached HEAD?

Решение - сделать ветку

git checkout -b issue-1
Switched to a new branch 'issue-1'

Граф коммитов

$ git log --graph --oneline --all
* a2ee9cb (issue-1) Breaking changes
| * c782f07 (HEAD -> master) Fixed typo
| * 31f9a86 Add important information
|/
* 953d1e0 Init commit

Слияние веток

$ git checkout master
Switched to branch 'master'

$ git merge issue-1
Auto-merging Readme.md
CONFLICT (content): Merge conflict in Readme.md
Automatic merge failed; fix conflicts and then commit the result.

Разрешение конфликтов

  1. Исправить файл вручную

  2. Добавить исправленные файлы в рабочую копию

    $ git add Readme.md

  3. Сделать коммит

    $ git commit -m "Fixed conflicts"
[master ce6210f] Fixed conflicts
 Committer: Ivan Belyavtsev <djbelyak@gmail.com>

После разрешения конфликта

$ git log --graph --oneline --all
*   ce6210f (HEAD -> master) Fixed conflicts
|\
| * a2ee9cb (issue-1) Breaking changes
* | c782f07 Fixed typo
* | 31f9a86 Add important information
|/
* 953d1e0 Init commit

Пометить коммит

$ git tag -a -m "Alpha version" v0.1

Список меток

$ git tag -l
v0.1
v1.0
